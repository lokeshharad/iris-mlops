VERSION: 1.2.0.3

VERSION FORMAT: MAJOR.MINOR.PATCH.BUILD

=== VERSION COMPONENT DESCRIPTIONS ===

MAJOR (1):
  - Represents major feature releases or significant architectural changes
  - Increment when there are breaking changes or complete overhauls
  - Current: 1 (OOP Trainer class with model selection and CV framework)

MINOR (2):
  - Represents new features, enhancements, or additions
  - Increment when adding new functionality without breaking existing code
  - Current: 2 (Added model selection with 5 candidate models: LR, DT, RF, SVC, KNN)

PATCH (0):
  - Represents bug fixes, performance improvements, and minor refinements
  - Increment for patches that don't introduce new features
  - Current: 0 (Latest stable release)

BUILD (1):
  - Represents internal build number or incremental improvements
  - Used for internal iterations, logging enhancements, and documentation updates
  - Current: 1 (Added comprehensive logging throughout pipeline)

=== VERSION HISTORY ===

v1.2.0.1 - Release
  Features:
    - OOP-based Trainer class for modular training pipeline
    - Cross-validation with StratifiedKFold (5-fold default)
    - Model selection across 5 candidate models:
      * Logistic Regression (with scaling)
      * Decision Tree
      * Random Forest (100 estimators)
      * SVC (with scaling)
      * KNN (with scaling)
    - Comprehensive metrics: Accuracy, Precision, Recall, F1 Score (weighted)
    - Classification reports for detailed performance analysis
    - Train-test split with stratification (80-20 split)
    - Joblib model serialization
    - Extensive logging at all pipeline stages

v1.2.0.2 - Release
  Features & updates:
    - Exposed CLI entrypoint `run_job.py` with argparse and operation dispatch
      * Operations: `train`, `single_predict`, `batch_predict`
    - `predict.py` refactored to `Predictor` class with `single_predict` and `batch_predict` support
    - `run_job.py` uses dictionary dispatch (no if/else in main)
    - Batch CSV support: `data/iris_batch_test.csv` with optional target column for evaluation
    - README updated with exact CLI examples and CSV format
    - Incremented build number to reflect documentation, CLI, and predict enhancements

v1.2.0.3 - Current Release
  Features & updates:
    - Model versioning: models saved with timestamped filenames in `models/`
    - `latest.txt` pointer file added to `models/` to indicate the most recent model
    - Training pipeline keeps only the latest 5 model files (old models auto-removed)
    - `Predictor.load_model` will automatically select the latest model when no explicit file is present
    - README updated to document versioning behavior

v1.1.0 - Proposed
  Features:
    - OOP Trainer class
    - CV and metrics evaluation
    - Basic model training

v1.0.0 - Initial
  Features:
    - Functional train script with basic RF classifier
    - Simple train-test split
    - Basic model serialization
