VERSION: 1.2.0.1

VERSION FORMAT: MAJOR.MINOR.PATCH.BUILD

=== VERSION COMPONENT DESCRIPTIONS ===

MAJOR (1):
  - Represents major feature releases or significant architectural changes
  - Increment when there are breaking changes or complete overhauls
  - Current: 1 (OOP Trainer class with model selection and CV framework)

MINOR (2):
  - Represents new features, enhancements, or additions
  - Increment when adding new functionality without breaking existing code
  - Current: 2 (Added model selection with 5 candidate models: LR, DT, RF, SVC, KNN)

PATCH (0):
  - Represents bug fixes, performance improvements, and minor refinements
  - Increment for patches that don't introduce new features
  - Current: 0 (Latest stable release)

BUILD (1):
  - Represents internal build number or incremental improvements
  - Used for internal iterations, logging enhancements, and documentation updates
  - Current: 1 (Added comprehensive logging throughout pipeline)

  Features:
